# コメント分析スキル

YouTube 動画・チャンネルのコメントを体系的に分析し、視聴者インサイトを抽出する。

## いつ使うか

- 特定の動画やチャンネルのコメントの傾向を知りたいとき
- 視聴者がどんな反応をしているか把握したいとき
- 改善要望やネガティブフィードバックを収集したいとき
- コメント欄から次の動画のアイデアを探したいとき

## いつ使わないか

- コメント数が少ない (10件未満) 動画の分析
- 再生数やエンゲージメント率など数値指標のみの分析 → video-performance を使う

## 手順

### Step 1: データ取得

MCP Server `samurai-youtube` からコメントデータを取得する。

**単一動画の分析**:
```
get_comments(video_id=対象動画ID, limit=100, sort_by=like_count, sort_order=desc)
```

**チャンネル全体の分析**:
```
get_comments(channel_id=対象チャンネルID, limit=200, sort_by=like_count, sort_order=desc)
```

**特定テーマの分析**:
```
get_comments(channel_id=対象チャンネルID, query=検索キーワード, limit=100)
```

取得時のポイント:
- `include_video_info: true` を指定し、どの動画へのコメントか分かるようにする
- いいね数でソートすると、共感を得ているコメントが上位に来る
- `stats` フィールドの `avg_likes`, `total_replies` も分析に活用する

### Step 2: 分析

取得したコメントに対して、以下の **5つの観点** で分析する。

#### 2-1. 感情分析 (Sentiment)

各コメントを以下の3カテゴリに分類する:

| カテゴリ | 判定基準 |
|---|---|
| ポジティブ | 称賛、感謝、共感、「勉強になった」「面白い」等 |
| ネガティブ | 批判、不満、失望、「つまらない」「間違ってる」等 |
| ニュートラル | 質問、情報共有、感想なし |

全体の感情分布 (ポジティブ率 / ネガティブ率) を算出する。

#### 2-2. テーマ分類

コメントを内容ベースでグルーピングする。典型的なテーマ:

- **内容への評価**: 動画の内容自体への感想
- **要望・リクエスト**: 「次は〇〇やってほしい」
- **質問**: 動画内容への質問
- **体験共有**: 視聴者自身の体験談
- **技術的フィードバック**: 音質、画質、編集への言及

#### 2-3. 要望・リクエスト抽出

「〜してほしい」「〜が知りたい」「次は〜」「〜の動画お願い」等のパターンを抽出し、リクエスト一覧を作成する。いいね数の多いリクエストほど需要が高い。

#### 2-4. 高エンゲージメントコメント

いいね数上位10件を抜き出し、なぜ共感を得ているかを分析する。
返信数が多いコメントはディスカッションを生んでいるため、特に注目する。

#### 2-5. 炎上・リスクシグナル

以下のパターンがあれば警告する:
- 同一テーマのネガティブコメントが集中
- 事実誤認の指摘
- 誤解を招く表現への批判
- 競合チャンネルとの比較批判

### Step 3: 出力

以下のフォーマットでレポートを出力する:

```markdown
## コメント分析レポート: [動画タイトル or チャンネル名]

### サマリー
- 分析対象: [コメント数]件 / [動画数]本
- 感情分布: ポジティブ [X]% / ニュートラル [Y]% / ネガティブ [Z]%
- 総合評価: [一言で]

### 感情分析
| カテゴリ | 件数 | 割合 | 代表的コメント |
|---|---|---|---|
| ポジティブ | X | X% | 「...」 |
| ネガティブ | X | X% | 「...」 |
| ニュートラル | X | X% | 「...」 |

### テーマ別分類
1. **[テーマ名]** (X件) - [概要]
2. **[テーマ名]** (X件) - [概要]
...

### 視聴者リクエスト TOP 5
| 順位 | リクエスト内容 | いいね数 | 動画 |
|---|---|---|---|
| 1 | ... | X | ... |

### 高エンゲージメントコメント
[上位コメントの引用と分析]

### リスクシグナル
[あれば記載、なければ「特になし」]

### 推奨アクション
1. [具体的アクション + 根拠]
2. [具体的アクション + 根拠]
3. [具体的アクション + 根拠]
```

## 注意事項

- コメントの感情分析は機械的な分類であり、文脈やニュアンスを完全には捉えられない
- 皮肉や冗談はポジティブ/ネガティブの誤分類が起きやすい
- コメント数が少ない場合 (30件未満)、統計的な傾向は参考程度に留める
- スパムコメントや宣伝コメントは分析対象から除外して判断する
