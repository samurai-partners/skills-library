# チャンネル横断比較スキル

管理対象の複数チャンネルを横断的に比較し、各チャンネルの特性と改善方向を明確にする。

## いつ使うか

- 複数チャンネルのパフォーマンスを比較したいとき
- 各チャンネルの強み・弱みを把握したいとき
- リソース配分の判断材料が必要なとき
- チャンネル全体のポートフォリオを俯瞰したいとき

## いつ使わないか

- 単一チャンネル・単一動画の詳細分析 → video-performance を使う
- 競合他社チャンネルとの比較 (DB 外のチャンネル)

## 手順

### Step 1: データ取得

**全チャンネル一覧**:
```
get_channels(sort_by=subscriber_count, sort_order=desc)
```

**各チャンネルの動画統計**:
```
get_videos(channel_id=各チャンネルID, limit=50, sort_by=views_total, sort_order=desc)
→ stats フィールドから views_sum, views_avg, views_median, engagement_rate_avg を取得
```

**チャンネル別の成長推移**:
```
get_snapshots(channel_id=各チャンネルID, start_date=90日前, granularity=weekly, include_deltas=true)
```

### Step 2: 分析

#### 2-1. 基本指標比較

| 指標 | 見方 |
|---|---|
| 登録者数 | チャンネルの規模 |
| 総再生数 | 累積パフォーマンス |
| 動画数 | コンテンツ量 |
| 登録者あたり再生数 | コンテンツの効率 (高い = 非登録者にもリーチ) |
| 動画あたり平均再生数 | 1本あたりの打率 |
| エンゲージメント率平均 | ファンの熱量 |

#### 2-2. コンテンツ効率マトリクス

各チャンネルを2軸でマッピング:

- **X軸**: 投稿頻度 (動画数 / 運用期間)
- **Y軸**: 動画あたり平均再生数

```
         高再生
           │
   効率的   │   理想的
   (少投稿  │  (多投稿
    高再生) │   高再生)
───────────┼───────────
   低調     │   多産低収
   (少投稿  │  (多投稿
    低再生) │   低再生)
           │
         低再生
     少←─投稿頻度─→多
```

#### 2-3. 成長トレンド比較

直近 90 日のスナップショットから:
- 週次再生数の推移を比較
- 成長率 (前月比) の比較
- 伸びているチャンネルと停滞しているチャンネルの特定

#### 2-4. コンテンツミックス

各チャンネルの動画構成を比較:
- ショート動画の割合
- 平均動画尺
- 投稿頻度
- タグ・キーワードの傾向

### Step 3: 出力

```markdown
## チャンネル横断比較レポート

### サマリー
- 分析対象: [チャンネル数]チャンネル
- 分析期間: [開始日] 〜 [終了日]
- 総合評価: [最もパフォーマンスが高いチャンネルと理由]

### 基本指標比較
| チャンネル | 登録者数 | 総再生数 | 動画数 | 平均再生数 | エンゲージメント率 |
|---|---|---|---|---|---|

### ポジショニングマップ
[投稿頻度 x 平均再生数の2軸マップ]

### チャンネル別所見
#### [チャンネル1]
- 強み: ...
- 弱み: ...
- 改善方向: ...

#### [チャンネル2]
...

### 成長トレンド
[週次推移の比較データ]

### リソース配分の推奨
1. [投資強化すべきチャンネル + 根拠]
2. [改善が必要なチャンネル + 具体策]
3. [維持でよいチャンネル + 理由]
```

## 注意事項

- 登録者数の差が大きいチャンネル同士を単純比較しない。規模を正規化した指標 (動画あたり再生数等) で比較する
- チャンネルのジャンルが異なる場合、エンゲージメント率の平均値が異なるのは自然
- 成長トレンドは少なくとも90日分のデータで判断する。短期の変動はノイズの可能性がある
